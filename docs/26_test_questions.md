# 営業支援AI テスト質問一覧

このドキュメントは、営業支援AIの性能を評価するためのテスト質問集です。
各質問に対する期待される動作と評価基準を記載しています。

---

## カテゴリA: 簡潔な情報照会

### A-1. 担当案件の確認
**質問**: 「私の担当案件は?」

**期待される動作**:
- `search_deals(sales_user_id=1)` のみを実行
- 案件リストを簡潔に箇条書きで表示（3-10行程度）
- 詳細なレポート形式は使わない

**評価基準**:
- [ ] 不要なツールを呼んでいないか（get_deal_details等）
- [ ] 簡潔に表示されているか
- [ ] レスポンス時間は10秒以内か

---

### A-2. ステージ別案件確認
**質問**: 「商談中の案件を教えて」

**期待される動作**:
- `search_deals(sales_user_id=1, deal_stage="商談")` を実行
- 該当案件を簡潔にリスト表示

**評価基準**:
- [ ] deal_stageパラメータを正しく指定しているか
- [ ] 簡潔な形式で回答しているか

---

### A-3. 失注案件確認
**質問**: 「失注案件を教えて」

**期待される動作**:
- `search_deals(sales_user_id=1, deal_stage="失注")` を実行
- 失注案件のリストを表示

**評価基準**:
- [ ] 正しいステージで検索しているか
- [ ] 簡潔に表示されているか

---

### A-4. 案件数確認
**質問**: 「商談中の案件数は?」

**期待される動作**:
- `search_deals(sales_user_id=1, deal_stage="商談")` を実行
- 件数を回答（例: 「3件です」）

**評価基準**:
- [ ] 件数のみを回答しているか
- [ ] 不要な詳細を含んでいないか

---

### A-5. 特定顧客の案件確認
**質問**: 「KDDIの案件状況は?」

**期待される動作**:
- `search_deals(customer_id=1)` または `search_customers(keyword="KDDI")` → `search_deals(customer_id=取得したID)` を実行
- KDDI関連の案件を簡潔に表示

**評価基準**:
- [ ] 顧客を正しく特定しているか
- [ ] 簡潔に表示されているか

---

## カテゴリC: 包括的な分析・レポート要求

### C-1. 担当案件のニュースまとめ
**質問**: 「私の担当案件の最新ニュースを調べて」

**期待される動作**:
1. `search_deals(sales_user_id=1)` で案件を取得
2. 主要な顧客企業を特定（金額上位3社など）
3. 各企業の `search_latest_news` を実行
4. レポート形式で整形

**評価基準**:
- [ ] ReAct形式で計画的にツールを実行しているか
- [ ] 金額上位企業を正しく特定しているか
- [ ] レポート形式で出力されているか
- [ ] データ取得日時が現在の日付になっているか

---

### C-2. 高額案件のニュースと戦略提案
**質問**: 「私の担当案件をリストアップして、そのうち金額が高い案件の最新ニュースを出して」

**期待される動作**:
1. `search_deals(sales_user_id=1)` で案件を取得
2. 金額でソート、上位企業を特定
3. 各企業の `search_latest_news` を実行
4. レポート形式で整形（企業別詳細分析含む）

**評価基準**:
- [ ] 計画的にツールを実行しているか
- [ ] 金額順にソートされているか
- [ ] レポート形式で出力されているか
- [ ] アクションプランが含まれているか

---

### C-3. 全体状況のまとめ
**質問**: 「担当領域の営業レポートを作成して」

**期待される動作**:
1. `get_user_info(user_id=1)` でユーザー情報取得
2. `search_deals(sales_user_id=1)` で案件を取得
3. 主要企業の `search_latest_news` を実行
4. 包括的なレポート形式で整形

**評価基準**:
- [ ] ユーザー情報を含んでいるか
- [ ] エグゼクティブサマリーが含まれているか
- [ ] 企業別詳細分析が含まれているか
- [ ] 戦略的提言が含まれているか

---

### C-4. 優先度付きアクションプラン
**質問**: 「今週の戦略を提案して」

**期待される動作**:
1. 案件情報を取得
2. ニュース情報を取得
3. 優先度を付けてアクションプランを提示

**評価基準**:
- [ ] 優先度（★★★等）が付いているか
- [ ] 短期・中期のアクションプランが含まれているか
- [ ] 戦略的提言が含まれているか

## 評価方法

### 各質問に対して以下をチェック:

1. **正確性**: 期待されるツールを正しく実行しているか
2. **効率性**: 不要なツールを呼んでいないか
3. **応答形式**: 質問タイプに応じた適切な形式で回答しているか
4. **レスポンス時間**: 許容範囲内で回答しているか
5. **データの新鮮さ**: 現在の日付（2026年）のデータを使用しているか

### スコアリング:
- ✅ 完璧: 5点
- ⚠️ 改善の余地あり: 3点
- ❌ 不適切: 0点

### 合格基準:
- 全体スコア: 80%以上
- カテゴリA: 90%以上（基本機能）
- カテゴリC: 70%以上（複雑な処理）
